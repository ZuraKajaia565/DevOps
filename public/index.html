Zura CI/CD Pipeline

A small Express application deployed to Vercel through GitHub Actions. The pipeline runs tests first, then deploys production only when the quality gate passes.

View health check GitHub Actions workflow
Production flow

Push to main → tests pass → Vercel deploys.

Pipeline overview

  1. Test Install dependencies and run Jest on each push or pull request.
  2. Deploy Deploy to Vercel only after tests pass on the main branch.
  3. Rollback Promote a previous stable Vercel deployment if production breaks.
